        Класс
Описывает только 1 объект
Имя класа простое и понятное. Должно описывать класс. Не глагол.
Четкий интерфейс.(описание) класс должен быть как черный ящик.
        Метод
Один метод - одна задача. Чем проще код - тем лучше.
Имя метода должно четко отображать его задачу(с маленькой буквы).
Нужно использовать все параметры метода. Не оставлять "на всякий случай".
Чем меньше параметров, тем лучше.
Не использовать параметры для хранения данных внутри метода.
Входящие данные по возможности проверять.
        Наследование
Подкласс - является более специлизированым типом суперкласса.
Лучше не делать множественное наследование.
		Инкапсуляция
Делать все элементы закрытыми, и потом открывать по мере необходимости
